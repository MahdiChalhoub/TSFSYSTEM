## How deploy_hotfix.sh Works

### Step 1: Version Bump
- Auto-generates version: `3.3.0-AG-YYMMDD.HHMM`
- Updates `src/lib/branding.ts` automatically

### Step 2: Sync from Production → Dev (preserves remote edits)
```bash
rsync -avz -e "ssh -i ~/.ssh/id_deploy" \
  --exclude 'node_modules' --exclude '.git' --exclude 'venv' \
  root@91.99.186.183:/root/TSFSYSTEM/ \
  /root/.gemini/antigravity/scratch/TSFSYSTEM/
```

### Step 3: Sync from Dev → Production (deploy changes)
```bash
rsync -avz -e "ssh -i ~/.ssh/id_deploy" \
  --exclude 'node_modules' --exclude '.git' --exclude 'venv' \
  /root/.gemini/antigravity/scratch/TSFSYSTEM/ \
  root@91.99.186.183:/root/TSFSYSTEM/
```

### Step 4: Database Migrations (on production)
```bash
ssh root@91.99.186.183 "docker exec tsfsystem-backend-1 python manage.py makemigrations"
ssh root@91.99.186.183 "docker exec tsfsystem-backend-1 python manage.py migrate"
```

### Step 5: Restart Backend Services
```bash
docker restart tsfsystem-backend-1 tsfsystem-celery_worker-1 tsfsystem-celery_beat-1
```

### Step 6: Rebuild Frontend (clean build in Docker)
```bash
docker-compose build --no-cache frontend
docker-compose up -d frontend mcp_agent_pulse
```

### Step 7: Restart Nginx Gateway
```bash
docker restart tsf_gateway
```

### Step 8: Cleanup
```bash
docker image prune -f
```

## Troubleshooting

### If deployment fails:
1. Check SSH connection: `ssh -i ~/.ssh/id_deploy root@91.99.186.183`
2. Check Docker status: `docker ps`
3. Check logs: `docker logs tsfsystem-frontend-1`
4. Restart manually: `docker-compose restart frontend`

### If site still shows old version:
1. Hard refresh browser (Ctrl+Shift+R)
2. Clear Cloudflare cache
3. Check Docker container is running: `docker ps | grep frontend`

### If 522 error:
1. Check SSL certificate on production
2. Verify Cloudflare SSL mode is "Full (strict)"
3. Check Nginx is running: `docker ps | grep gateway`
