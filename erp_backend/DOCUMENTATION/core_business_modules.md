# DOCUMENTATION: Core Business Modules

## Overview
This document covers the core business logic of the Dajingo ERP, restored as part of the v1.2.6-b704 update.

---

## 1. Finance & Ledger Workflow
**Goal**: Provide a compliant, double-entry accounting system with multi-tenant isolation.
**Data Source (READ)**: `ChartOfAccount`, `FiscalYear`, `FiscalPeriod`, `JournalEntry`.
**Data Target (SAVE)**: `JournalEntry`, `JournalEntryLine`.
**Workflow**:
1. `LedgerService.create_journal_entry` is called with debit/credit lines.
2. The system verifies if the total is balanced and within an open `FiscalPeriod`.
3. If `POSTED`, `LedgerService.post_journal_entry` updates the `balance` fields in `ChartOfAccount`.
4. Reports (Balance Sheet, P&L) are generated by rolling up balances through the account hierarchy.

---

## 2. Inventory Management
**Goal**: Track stock levels, valuations, and movements across multiple warehouses.
**Data Source (READ)**: `Product`, `Warehouse`, `Inventory`.
**Data Target (SAVE)**: `Inventory`, `InventoryMovement`.
**Workflow**:
1. `InventoryService.receive_stock` increases quantity and calculates the new **Average Moving Cost (AMC)**.
2. `InventoryService.reduce_stock` decreases quantity and captures the current AMC for COGS booking.
3. Every movement creates an `InventoryMovement` record for auditing.
4. Valuation is calculated in real-time by summing `quantity * cost_price`.

---

## 3. POS (Point of Sale)
**Goal**: Facilitate fast retail transactions with automatic accounting and inventory updates.
**Data Source (READ)**: `Product`, `Warehouse`, `FinancialAccount`.
**Data Target (SAVE)**: `Order`, `OrderLine`, `JournalEntry`, `Inventory`.
**Workflow**:
1. Items are scanned and added to a checkout session.
2. `POSService.checkout` atomicly:
   - Creates a 'SALE' `Order`.
   - Reduces stock for each item via `InventoryService`.
   - Records a `JournalEntry` (Dr Cash, Cr Revenue, Cr VAT, Dr COGS, Cr Inventory).
3. Prints/Returns the finalized invoice data.

---

## 4. MCP (Mission Control Protocol)
**Goal**: Provide AI-driven analysis and tool execution within the ERP.
**Data Source (READ)**: Internal APIs via `ConnectorEngine`.
**Workflow**:
1. User provides a prompt to the AI.
2. The AI uses `MCPService` to identify required tools.
3. Tools execute internal requests (e.g., "Get stock for X") routed through `ConnectorEngine` to ensure multi-tenant security.
